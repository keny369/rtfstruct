rtfstruct

A Python library for turning RTF into structured, AI-ready document data.

Read RTF as structure, not just text.

rtfstruct parses Rich Text Format into a neutral document AST that preserves paragraphs, inline styles, links, fields, notes, annotations, lists, tables, images, metadata, source spans, and recoverable diagnostics.

The AST can then be exported to JSON for machines, Markdown for humans, or deterministic RTF for roundtrip workflows.

rtfstruct was written by Lee Powell, who built the production C++ RTF parser and writer used in Scrivener for Windows and Linux—pressure-tested for more than a decade across hundreds of thousands of writers and long-form projects. That engineering was later sold as a white-label engagement to Literature & Latte.

Today he advises executives and boards on structural AI readiness through Lumen & Lever: establishing control over AI already in use, clarifying exposure, and deciding whether an organization is structurally ready to scale before more capital goes out the door—the same discipline he applied as architect of Scrivener and Scapple and as an enterprise architect at Commonwealth Bank and Deutsche Bank.

rtfstruct applies that production-informed mindset to document ingestion for AI pipelines, not only to board-level AI architecture.

Most AI document pipelines fail before retrieval begins. Structure is flattened. Tables lose meaning. Lists collapse. Comments and notes disappear. Source traceability is broken. The model is then asked to reason over damaged input.

rtfstruct is built for the layer most pipelines underestimate: document ingestion before chunking, embedding, retrieval, and generation.

It is not a plain-text stripper, a Markdown converter, or a toy parser. It is an AST-first RTF reader and writer for systems where document structure still matters: AI ingestion, RAG, legal discovery, banking archives, forensic document tracing, publishing, and long-form document intelligence.

Quick Start

Install, parse, inspect, export.

The core runtime has no heavyweight document-processing dependencies. Development extras install pytest and Sphinx for validation and documentation.

Install for Development

python -m pip install -e ".[dev,docs]"
python -m pytest -q
sphinx-build -E -b html docs docs/_build/html

Use from Python

from rtfstruct import ParserOptions, parse_rtf

document = parse_rtf(
    r"{\rtf1\ansi Hello, \b world\b0!}",
    options=ParserOptions(track_spans=True),
)

print(document.to_json())
print(document.to_markdown())
print(document.to_rtf())
`rtfstruct` is useful when RTF needs to become structured data for AI ingestion, document intelligence, validation pipelines, legal tooling, local conversion, or deterministic roundtrip tests.

Parser Internals

How the reader understands RTF.

RTF is a nested control-word language. The parser keeps interpretation explicit: tokenization is separate from semantic dispatch, and mutable parser state owns destinations, styles, tables, lists, notes, annotations, images, and diagnostics.

Lexer Tokenization

flowchart TD A[Raw RTF text] --> B{Current char} B -->|{| C[GROUP_START token] B -->|}| D[GROUP_END token] B -->|backslash + letters| E[CONTROL_WORD with optional integer parameter] B -->|backslash + apostrophe + hex| F[HEX_CHAR token] B -->|backslash + symbol| G[CONTROL_SYMBOL token] B -->|plain chars| H[TEXT token] C --> I[Preserve source offsets] D --> I E --> I F --> I G --> I H --> I

Destination Routing

flowchart TD A[Control word enters dispatch] --> B{Destination or property?} B -->|fonttbl colortbl info| C[Redirect text into metadata tables] B -->|field fldinst fldrslt| D[Field context stack] B -->|footnote| E[Footnote context] B -->|annotation| F[Annotation context] B -->|pict| G[Image context] B -->|listtable listoverride| H[List definition context] B -->|trowd cell row| I[TableBuilder context] B -->|style controls| J[Flush text and update current style] B -->|paragraph controls| K[Snapshot paragraph style] C --> L[Document metadata/resources] D --> M[Link or Field inline node] E --> N[FootnoteRef + document footnote] F --> O[AnnotationRef + document annotation] G --> P[ImageInline + document image] H --> Q[List post-pass metadata] I --> R[Resolved Table AST] J --> S[TextRun style] K --> T[Paragraph style]
Unicode Recovery

`\uN` values use signed 16-bit recovery, skip fallback bytes according to `\ucN`, and emit replacement characters plus diagnostics for malformed values.

Hex and Codepages

`\'xx` bytes decode through the active font charset, document `\ansicpgN`, or a CP1252 fallback with diagnostics for unsupported codepages.

Diagnostics

Recoverable issues are capped and deduplicated, then attached to `Document.diagnostics` for machine-readable inspection.

AST Contract

The document model.

The AST is the stable bridge between parsing and conversion. It avoids Qt, Markdown, and RTF-specific output concerns while retaining document structure and semantic formatting.

Conversion Recipes

Read once, export many ways.

`rtfstruct` separates parsing from output. The same `Document` AST can produce structured JSON, readable Markdown, deterministic RTF, CLI output, or integration adapter shapes.

RTF to JSON

from rtfstruct import JsonOptions, parse_rtf

document = parse_rtf(rtf_bytes)
data = document.to_json(
    options=JsonOptions(
        include_diagnostics=True,
        include_image_data=False,
    )
)

RTF to Markdown

from rtfstruct import MarkdownOptions, parse_rtf

document = parse_rtf(rtf_text)
markdown = document.to_markdown(
    options=MarkdownOptions(
        preserve_colors=True,
        complex_tables="html",
    )
)

AST to RTF

from rtfstruct import parse_rtf, to_rtf, write_rtf

document = parse_rtf(r"{\rtf1 Hello}")
rtf = to_rtf(document)
write_rtf(document, "normalized.rtf")

Integration Helpers

from rtfstruct.integrations import (

markdown = convert_rtf_to_markdown(rtf_text)

API Reference

Public Python surface.

Anything exported from `rtfstruct.__init__` is public API for the pre-alpha package. Internal modules may continue to evolve.

Reader

parse_rtf(data, options=None)
Parse bytes or text into a `Document`.

read_rtf(path, options=None)
Read an RTF file and parse it.

Writers

to_rtf(document)
Return deterministic RTF text.

write_rtf(document, path)
Write RTF to a file.

to_markdown(document, options=None)
Convenience Markdown export.

Options

ParserOptions: recovery, image extraction, spans, safety caps.

JsonOptions: null handling, diagnostics, image payloads.

MarkdownOptions: colors, fonts, annotations, complex tables.

Core AST

Document, Paragraph, TextRun, TextStyle, ParagraphStyle, StyleSheet.

Structure

ListBlock, ListItem, Table, TableCell, Image, ImageInline.

Semantics

Link, Field, Footnote, FootnoteRef, Annotation, AnnotationRef.

Inspect Diagnostics

from rtfstruct import parse_rtf

document = parse_rtf(r"{\rtf1\u999999?}")

for diagnostic in document.diagnostics:
    print(
        diagnostic.severity,
        diagnostic.code,
        diagnostic.message,
    )

Semantic Roundtrip

from rtfstruct import parse_rtf, to_rtf

original = parse_rtf(rtf_text)
reparsed = parse_rtf(to_rtf(original))

assert original.semantic_equals(reparsed)
Command Line

Convert files without writing Python.

The `rtfstruct` console script is installed from package metadata and can write to stdout or an output file. 

rtfstruct input.rtf --to markdown > output.md
rtfstruct input.rtf --to json --output output.json
rtfstruct input.rtf --to diagnostics
rtfstruct input.rtf --to rtf --output normalized.rtf
rtfstruct input.rtf --to json --track-spans --include-image-data
json markdown rtf diagnostics source spans image payloads
